# Provenance And Privacy Policy

Latest local workflow material reviewed: July 2, 2026.

## Source Rights

Use only materials the instructor, teaching team, or rights holder is authorized
to process. Open educational resource status requires an explicit open license
or public-domain status. Course access alone is not a publication license.

## Student Privacy

Exclude student submissions, grades, rosters, identifying student comments, raw
recording metadata, passcodes, and any transcript text that has not been
de-identified and approved for the intended use.

If a transcript is allowed, preserve timestamps for evidence while replacing
student names and incidental personal information.

## Video Boundary

Do not publish video, audio, recording links, passcodes, or downloaded media.
If a future workflow uses screen-only frames for evidence, it must mask faces,
names, chat, and platform metadata before use, and it must record that policy in
the manifest.

## Attribution

The professor's teaching materials are sources, not public assets included in
this repository. Generated drafts should identify who is responsible for the
draft, which source set was used, and which review layers have actually passed.

Do not imply university, instructor, platform, or publisher endorsement without
written approval.

## Verification Claims

Use precise labels:

- `V_det`: deterministic verification completed;
- `V_LLM`: model critique completed;
- `V_human`: human review completed.

Do not claim a book is release-grade because one verification layer passed.
Render health and source-grounding checks are not legal, copyright, privacy, or
pedagogical approval.
